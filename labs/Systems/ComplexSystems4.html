Small-World Networks (Watts–Strogatz)

short paths · high clustering

Idea: Start with a ring where each node connects to its k nearest neighbors. Then randomly rewire a fraction of edges. You get short average path lengths (like random graphs) but keep high clustering (like regular lattices).

Construction sketch:

  • Nodes on a ring: i = 0…N−1.
  • Each node connects to k/2 neighbors on each side.
  • For each edge, with probability p, rewire the far end to a random node.

You can tune p from 0 (regular lattice) to 1 (random graph) and see the “small-world” regime emerge in between.

Real-world links:

  • Power grids, roads, airline routes, social networks often show small-world properties.
  • Impacts how fast failures, ideas, or infections travel.
  • Useful mental model for Networks: local clustering + a few long links.

SIR Epidemic on a Graph

compartments · per-edge infection

Idea: Classic SIR model, but instead of a well-mixed population we constrain infections to the network edges.

Well-mixed SIR equations:

dS/dt = −β S I
dI/dt = β S I − γ I
dR/dt = γ I

On a graph, we think more discretely:

  • Each node is S, I, or R.
  • At each step, for each infected node, neighbors become infected with probability pinf.
  • Infected node recovers with probability prec.

Real-world links:

  • Computer viruses spreading through network connections.
  • Contagion in power grids (overloads), not just literal diseases.
  • Helps see why “who is connected to whom” matters more than just counts.

Cascading Failures

threshold dynamics · overload

Idea: Each node carries some load and has a capacity. If load > capacity, the node fails and pushes its load to neighbors. Neighbors can then exceed capacity and fail too → cascade.

A simple threshold abstraction:

  • Each node i has state: normal, overloaded, or failed.
  • If a sufficient fraction of neighbors are failed or overloaded, node i becomes overloaded.
  • Overloaded → failed in the next step.

More detailed models track actual flows F and capacities C, e.g. Fᵢ(t+1) = Fᵢ(t) + Σⱼ ΔFⱼ→ᵢ and fail when Fᵢ > Cᵢ.

Real-world links:

  • Power grids (line trips, re-routed power, further trips).
  • Network congestion and overload in data networks.
  • Industrial alarms cascades: one failure triggers loads / modes that break others.

Consensus / Opinion Dynamics (DeGroot)

distributed averaging · agreement

Idea: Each node holds a scalar value (opinion, estimate, voltage setpoint). At each step, it averages its own value with neighbors' values → eventually they converge.

Basic DeGroot update:

xᵢ(t+1) = (1 − α) xᵢ(t) + α · average{xⱼ(t) : j is neighbor of i}

  • α – how much weight you give to neighbors.
  • In matrix form: x(t+1) = W x(t) with W a row-stochastic matrix.

Real-world links:

  • Sensor fusion: distributed sensors agreeing on a measurement.
  • Clock / phase synchronization in distributed systems.
  • Multi-agent control: drones agreeing on a formation center / heading.

Mapping to my projects

Unit 4 is the “network brain” you can put under everything else:

  • PLCs, Industrial Controllers: obvious fit — cascading failures, routing, consensus on controllers.
  • Log analysis: think of logs as signals on a network of components; cascades in logs = cascades in topology.
  • Trending apps: you can literally render these graphs as glowing grids where infections, failures, and consensus waves propagate.

Big mental move: instead of “A, B, C are broken”, ask “what graph do they live in, and what regime (stable / epidemic / cascade / consensus) is this graph in right now?”
